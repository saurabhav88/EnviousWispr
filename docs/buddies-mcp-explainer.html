// registered tools
7 MCP Tools
All tools are registered in main.py and available to Claude Code via the MCP stdio protocol. Click any tool to expand its details.
1
buddies()
Send message and get LLM response
▾
Arguments
session: str           # Session name (auto-creates)
message: str           # User message to send
provider: str          # "gemini" or "openai"
model: str              # Model ID
system_prompt: str?    # Optional custom prompt
Behavior (ordered steps)
  1. Look up session by name, create if not found
  2. Merge default + custom system prompts, store in session
  3. Append user message to conversation history
  4. Call provider API with full history + system prompt
  5. Append assistant response to history
  6. Atomically save session to disk
  7. Return response to Claude Code
⚡ Rollback on failure — user message removed if LLM call fails
2
buddies_inject()
Inject context without triggering LLM
▾
Arguments
session: str     # Session name
content: str     # Content to inject (files, docs)
label: str      # Human-readable label for this context
Behavior
  1. Appends labeled content block to message history
  2. Does NOT call the LLM — no API cost
  3. Saves updated session to disk
Use for priming sessions with large context before first message
3
buddies_set_system_prompt()
Update system prompt for existing session
▾
Arguments
session: str              # Session name
system_prompt: str     # New system prompt text
Behavior
  1. Updates the system_prompt field in session metadata
  2. Takes effect on the next buddies() call — not retroactive
  3. Saves updated session to disk
Stored in session.metadata.system_prompt
4
buddies_get_history()
View conversation history
▾
Arguments
session: str      # Session name
last_n: int?     # Limit to last N messages (optional)
Behavior
  1. Loads session from store (lazy — full content on first access)
  2. Returns message history, optionally sliced to last N
  3. Includes role, content, and timestamps for each message
Read-only — does not modify session or trigger any API call
5
buddies_list_sessions()
List all sessions with metadata
▾
Arguments
(none)
Returns per session
name: str             # Session identifier
provider: str        # "gemini" | "openai"
model: str            # Model ID
message_count: int   # Total messages
created_at: str      # ISO timestamp
Behavior
  1. Uses lazy-loaded metadata index (no full content reads)
  2. Metadata indexed at server startup, kept in memory
6
buddies_reset()
Clear messages, keep session config
▾
Arguments
session: str  # Session name
Behavior
  1. Clears the message history array to empty
  2. Preserves: name, provider, model, system_prompt
  3. Saves updated session to disk atomically
Config survives reset — only messages are cleared
7
buddies_delete()
Permanently delete session
▾
Arguments
session: str  # Session name
Behavior
  1. Removes session from in-memory store
  2. Deletes JSON file from disk permanently
  3. Removes from metadata index
Irreversible — no undo

// engineering decisions
Design Patterns
Four key architectural decisions that make Buddies reliable, efficient, and concurrent-safe.
💤
Lazy Loading
Session metadata is indexed at server startup. Full message content is only loaded from disk on first access — keeps startup fast regardless of session count.
# At startup: only index (no full reads)
for f in glob("sessions/*.json"):
  meta = read_metadata_only(f)
  self._index[meta.name] = meta

# Full load deferred to first access
async def load(self, name):
  if name not in self._loaded:
    self._loaded[name] = read_full(name)
⚛️
Atomic Writes
Sessions are written to a temp file first, then renamed. os.rename() is atomic on POSIX — a crash mid-write never leaves a corrupt session file.
# Write to temp, then atomic rename
tmp = path + ".tmp"
async with aiofiles.open(tmp, "w") as f:
  await f.write(json.dumps(data))

# Atomic swap — old file replaced instantly
os.rename(tmp, path)           # ← atomic
🔒
Per-Session Async Locks
Each session has its own asyncio.Lock. Concurrent calls to the same session are serialized — no race conditions on message history or disk writes.
# One lock per session, created lazily
self._locks: dict[str, asyncio.Lock] = {}

def _lock(self, name):
  if name not in self._locks:
    self._locks[name] = asyncio.Lock()
  return self._locks[name]

async with self._lock(session_name):
  # safe: append + write
🏭
Provider Singletons
One httpx.AsyncClient per provider type, shared across all sessions. Connection pooling, keep-alive, and proper lifecycle management — no per-call client overhead.
# providers/__init__.py — registry singleton
_registry: dict[str, LLMProvider] = {}

def get_provider(name: str) → LLMProvider:
  if name not in _registry:
    _registry[name] = _create(name)
  # reuses existing httpx client
  return _registry[name]
// failure handling
Rollback on Failure
Normal flow (LLM succeeds)
1. Append user message → history
↓
2. Call LLM API ← success ✓
↓
3. Append response → save to disk
Failure flow (LLM errors)
1. Append user message → history
↓
2. LLM API fails ✗ (quota/network)
↓
3. Roll back → remove user message
