# The Developer

## Who they are
Software engineers, DevOps folks, and technical leads who write code all day but also need to produce a surprising amount of prose — PR descriptions, documentation, Slack threads, incident reports, and design docs.

## Pain points
- Context-switching from code to prose is jarring and breaks flow state
- Writing PR descriptions, code reviews, and documentation feels tedious compared to the actual coding
- RSI from 8+ hours of daily typing across keyboard and trackpad
- Cloud-based dictation tools are a non-starter — they don't want audio of their work environment hitting someone else's servers

## How they'd use EnviousWispr
- Dictate commit messages and PR descriptions without leaving the terminal workflow
- Draft Slack replies and code review comments by voice, then let LLM post-processing clean up the filler
- Write technical documentation and READMEs by talking through the architecture — Standard preset cleans up grammar, Formal preset adds polish
- Switch between writing styles depending on context — terse for terminal notes, full sentences for Notion docs. Per-app presets (automatic switching) are on the roadmap.

## Keywords they search for
- "voice to text for developers mac"
- "dictation app that works offline privacy"
- "speech to text macOS open source"
- "RSI programmer voice typing"
- "whisper local transcription mac app"

## Tone shift
Technical and direct. Show terminal screenshots, mention the tech stack (WhisperKit, CoreML, Apple Silicon). Respect their intelligence — skip the hand-holding and show how it fits into their existing workflow. Mention that it's open source early and often.
