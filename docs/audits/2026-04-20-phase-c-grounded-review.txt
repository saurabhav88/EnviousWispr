9. The merge algorithm is acceptable for the intended live-dictation path, but it is not a true global newest-first sort. Because `TranscriptStore.loadAll()` sorts `diskRows` by `createdAt` descending and `append(_:)` inserts at index 0, normal app-generated transcripts will stay newest-first in practice. But `inFlightRows + diskRows` can misorder data if an in-flight append has a `createdAt` older than the newest disk row. A realistic scenario is a future-dated or manually imported legacy JSON already on disk; a new live transcript appended during load would still be forced to the front even if its `createdAt` is earlier. So the algorithm preserves “append order first,” not strict global `createdAt` ordering across arbitrary data.
