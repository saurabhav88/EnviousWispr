# Recording sound provenance

All 24 WAV files (12 pairings × start/stop: `dustMote_*`, `velvetHush_*`, `mutedConfirm_*`, `whisperTick_*`, `roundPebble_*`, `paperTap_*`, `softHush_*`, `lowNod_*`, `cloudPop_*`, `velvetTap_*`, `satinShift_*`, `airGlint_*`) were generated procedurally for EnviousWispr using additive sine/triangle synthesis, filtered noise, and amplitude envelopes. They contain no sampled, recorded, licensed, Apple, Slack, Discord, or WisprFlow audio.

Generated by `scripts/generate-recording-sounds.py`; running that script in a clean directory with only a stock Python 3 install (standard library only, no numpy/scipy/pip install step) reproduces these files run-to-run (verified via a hash-run-hash-diff check, `scripts/generate-recording-sounds.py` docstring). Every noise-based call uses a fixed seed, so re-running the script is deterministic even though the synthesis includes randomness; only the ORIGINAL 2026-07-17 migration off numpy for the first 4 pairings changed the noise-based Cloud Pop's bytes (documented in #1342), everything generated since is stable run-to-run.

Ref: issue #1342 (original 4 pairings, 2026-07-17), issue #1618 (8 more pairings + reordering, 2026-07-17).
