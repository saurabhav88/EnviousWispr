# The Accessibility User

## Who they are
People with RSI, carpal tunnel syndrome, motor disabilities, chronic pain conditions, or visual impairments for whom traditional typing is painful, exhausting, or impossible. Voice input isn't a convenience — it's a necessity.

## Pain points
- Typing causes physical pain that worsens over time, forcing difficult choices about how to spend limited typing capacity
- Existing accessibility tools are clunky, cloud-dependent, or require complex setup that itself demands extensive typing
- Cloud-based voice tools raise serious privacy concerns, especially when dictating medical information, therapy notes, or personal health details
- Many dictation tools produce output so inaccurate it takes more effort to correct than to type in the first place

## How they'd use EnviousWispr
- Use as their primary text input method across all applications — email, chat, documents, web forms
- Write code by voice using writing style presets — Standard for clean output, Friendly for casual notes. Per-app presets are on the roadmap.
- Produce long-form writing (reports, personal projects, correspondence) without the physical toll of typing
- Rely on on-device processing to keep sensitive medical and personal dictation completely private

## Tone shift
Respectful and matter-of-fact. Never inspirational or performative about disability. Focus on reliability, accuracy, and the practical reality that this tool needs to work every time because there's no fallback. Emphasize on-device privacy as non-negotiable, not a bonus feature. Acknowledge that setup and configuration should require minimal typing. Never treat accessibility as a niche afterthought — position it as a core use case.
