Brain System Evolution

How the agent knowledge layer grew from hand-maintained markdown to a partially self-healing, integrity-checked system. V0 through V2.1.

V0 The Starting Point — Pure Hand-Maintenance
What existed
  • □ Hand-written markdown in .claude/knowledge/ — architecture.md, gotchas.md, conventions.md, and others
  • □ Agent files in .claude/agents/ and skill files in .claude/skills/
  • □ All facts — file counts, type lists, line numbers, dependency versions — written and updated entirely by hand
The pain
pain File counts, type lists, and line numbers went stale the moment code changed
pain No way to know if a knowledge file was outdated without manually checking every source file
pain Dependency versions hardcoded in three agent files and distribution.md — diverged silently over time
pain New sessions started with potentially stale context — the AI would confidently state wrong facts
V1 Generated Files + Session Priming
What was built
  • + brain-refresh.sh — auto-generates 4 files from source: file-index.md, type-index.md, task-router.md, feature-catalog.md
  • + brain-check.sh — diffs generated files against source to detect staleness
  • + brain-prime.sh — runs at session start, auto-fixes stale generated files before any agent reads them
  • + Manual/generated split via <!-- MANUAL SECTION --> delimiter — human prose preserved across regeneration
Pain solved
solved Code-derived facts (file counts, type lists, line counts) can no longer go stale — regenerated on every session
solved Session start always has fresh derived data via brain-prime.sh
Pain remaining
gap Canonical files (architecture.md, gotchas.md, whisperkit-research.md) had zero staleness detection — still 100% hand-maintained
gap When WhisperKit highway shipped: gotchas.md missed 6 traps, whisperkit-research.md contradicted the codebase, architecture.md missed a protocol conformer
V2 Auto-Sections + Source Annotations + Integrity Checks
Three new mechanisms
1
Auto-generated sections in canonical docs
<!-- BEGIN AUTO: name --> / <!-- END AUTO: name --> markers. brain-refresh.sh replaces content between markers; human prose outside is preserved. Initially 2 sections: a protocol_conformers table in architecture.md and a whisperkit_defaults table in whisperkit-research.md.
2
BRAIN ID source annotations
// BRAIN: gotcha id=silence-padding in Swift source files linked bidirectionally to <!-- BRAIN: id=silence-padding --> in docs. brain-check.sh verifies every source annotation has a matching doc entry. 8 annotations across 5 source files.
3
Agent and skill file integrity
brain-check.sh verifies: all agent refs in CLAUDE.md point to existing files, all skill refs in agent files point to an existing SKILL.md, and orphan detection runs for both directions.
Pain solved
solved Machine-derivable facts in canonical docs auto-update — no longer fully hand-maintained
solved Source gotchas have verifiable links to doc entries — a source annotation with no doc match fails brain-check.sh
solved Broken agent/skill cross-references are caught automatically rather than discovered at runtime
Pain remaining
gap Only 2 auto-sections — most machine-derivable facts still scattered across agent files as hardcoded text
gap Dependency versions duplicated in 3 agent files and distribution.md — still no single authoritative source
gap Adding a settings tab required manually updating both architecture.md and wispr-eyes.md
V2.1 Deep Coverage + Deduplication
What was built
  • + 6 new auto-generators added to brain-refresh.sh — total now 8 auto-sections plus the original 4 fully-generated files
  • + 9 auto-sections across 4 files (up from 2 sections in 2 files):
    • architecture.md → settings_sections, pipeline_states, llm_providers, asr_backend_types, audio_constants, protocol_conformers
    • whisperkit-research.md → whisperkit_defaults
    • distribution.md → dependency_versions
    • wispr-eyes.md → settings_sections
  • + 3 agent files deduplicated — hardcoded dependency versions replaced with references to canonical source in distribution.md
  • + brain-check.sh expanded to validate all new auto-section files and coverage
Pain solved
solved Adding a settings tab, LLM provider, or ASR backend now auto-propagates to all relevant docs — no manual hunting
solved Dependency version bumps auto-update in distribution.md and propagate to agent files that reference it
solved "One file says X, another says Y" class of bugs is structurally eliminated for all covered facts
What's left
minor Prose sections in canonical docs (strategic rationale, usage guidance) still hand-maintained — intentional, not a bug
minor BRAIN annotation coverage is partial — only 8 gotcha annotations; not every non-obvious code behavior has one

Remaining exposure
  • ● Strategic and prose content in canonical docs is still entirely human-maintained. This is correct — the system is not trying to auto-generate reasoning or design intent, only machine-derivable facts.
  • ● BRAIN annotation coverage is incomplete. Any non-obvious behavior in Swift source that hasn't been annotated still has no guaranteed doc link. Incremental: add annotations when bugs reveal a gap.
  • ● New auto-sections require a dev to add the generator to brain-refresh.sh and the marker to the target doc. There's no automatic discovery of "this fact should have an auto-section."
