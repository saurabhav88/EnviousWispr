# Internal Links Map

Pages available for internal linking in blog posts.

| Page | URL | When to link |
|------|-----|-------------|
| Homepage | / | General product references |
| How It Works | /how-it-works/ | Technical explanations, pipeline details |
| Blog Index | /blog/ | "Read more on our blog" references |
| Welcome Post | /blog/welcome-to-enviouswispr/ | Origin story, product intro |
| GitHub Releases | https://github.com/saurabhav88/EnviousWispr/releases | Download CTAs |
| GitHub Repo | https://github.com/saurabhav88/EnviousWispr | Open source references, contributing |

## Linking Guidelines

- Every blog post should link to at least 2 internal pages
- Always link to /how-it-works/ when explaining the transcription pipeline
- Always link to GitHub releases for download/install CTAs
- Link to other blog posts when topically relevant (check ../website/src/content/blog/ for existing posts)
- Use descriptive anchor text, not "click here"
